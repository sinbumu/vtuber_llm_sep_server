# MVP_BACKLOG.md — 우리 프로젝트 기준 분류

전제: **Unity 오버레이 + Python 백엔드** 구조.

---

## MVP (반드시)

- 음성 대화 파이프라인(마이크 → STT → LLM → TTS)  
  이유: 제품 정체성의 핵심
- WebSocket 기반 실시간 통신  
  이유: 저지연 대화에 필요
- 캐릭터 기본 표시(모델 로드/표정 반영)  
  이유: 캐릭터형 UX의 최소 요건
- 대화 로그 저장/불러오기  
  이유: 사용자 유지/연속성
- 엔진 교체 설정(UI or 파일)  
  이유: 환경별 설치/품질 대응

---

## Nice-to-have

- 펫 모드 전용 레이아웃(투명/항상 위/클릭 통과)  
  이유: 데스크톱 UX 강화
- 내면 생각/무음 텍스트 표시  
  이유: 캐릭터 연출 강화
- 기본 프리셋/캐릭터 전환  
  이유: 사용성 개선

---

## 심화/실험

- 시각 인식(카메라/스크린/스크린샷)  
  이유: 권한/보안 리스크가 큼
- MCP 도구 연동(웹검색/도구 사용)  
  이유: 공격 표면 증가, 권한 설계 필요
- 음성 번역(대화 언어/음성 언어 분리)  
  이유: 복잡도와 비용 증가

## 제외(현재 단계)

- Live2D 샘플 모델 포함 배포  
  이유: 상용 라이선스 리스크
- 복잡한 방송/라이브 연동 기능  
  이유: MVP 범위 과도
