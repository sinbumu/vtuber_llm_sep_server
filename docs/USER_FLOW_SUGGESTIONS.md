# USER_FLOW_SUGGESTIONS.md — 사용자 플로우 제안

이 문서는 디자이너가 **와이어프레임**을 그릴 수 있도록,
핵심 사용자 흐름 3가지를 단계별로 정리합니다.

---

## 1) 첫 실행 온보딩

1. **환영 화면**
   - 앱 소개 + “시작하기” 버튼
2. **모드 선택**
   - 웹 모드 / 데스크톱 펫 모드 (설명 카드)
3. **기본 프리셋 선택**
   - 캐릭터 프리셋 목록 + 미리보기
4. **권한 요청**
   - 마이크 권한 요청
5. **LLM 연결 테스트**
   - “연결 테스트” 버튼 + 성공/실패 안내
6. **대화 시작**
   - 마이크 입력 표시 + 첫 인사/튜토리얼 팁

---

## 2) 캐릭터/오버레이 설정 변경

1. **설정 진입**
   - 상단 메뉴 → 캐릭터/디스플레이
2. **캐릭터 선택**
   - 모델 카드 + 이름/설명
3. **표정/감정 맵**
   - 감정 태그 목록 + 미리보기
4. **오버레이 설정**
   - 투명도, 크기, 위치, 항상 위 토글
5. **저장 & 적용**
   - “적용하기” + 결과 미리보기

---

## 3) LLM/프롬프트/메모리 설정 변경

1. **설정 진입**
   - 상단 메뉴 → AI/대화
2. **LLM 엔진 선택**
   - Ollama/클라우드 등 드롭다운
3. **모델/키 입력**
   - 모델명, API Key, 베이스 URL
4. **프롬프트 편집**
   - 캐릭터 프롬프트 편집기
5. **테스트/저장**
   - “테스트 질문” + “저장”

---

## 권한 요청 UX 팁

- 마이크 권한은 **온보딩 중 한 번만** 요청  
- 거부 시 **대체 입력(텍스트)** 안내  
- 권한 실패 시 재요청 버튼 제공
